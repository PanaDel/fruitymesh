Testing Module (moduleId 12)

Purpose

The Testing Module is designed to automate tests for battery and throughput measurements. The module will allow us to measure the battery consumption of each individual application and also aid in estimating the throughput of the network.

Functionality

The Testing Module has two roles.

  • Master: Testing Module by default starts in test role "Master". Master node initialises the tests by sending a start test message to the node which will perform tests.

  • Slave: Testing Module changes its role to slave when it receives a message to start test.

Each test runs for a maximum of 30 secs. Master node initialises every test by first connecting to the slave node and then sending a message to start test. On the other hand slave node acknowledges this start test message, disconnect from partner and switch LEDs on for 2 sec to indicate the start of test.

Test Applications

The applications currently being tested are Advertisement, Scanning, LED, Latency and Connection Tests.

Detailed description of these application are given below.

LED Test (Application Id 1)

Slave node switch red LED on for 10 sec and do nothing and then switch green LED on and do nothing.

Advertisement Test (Application Id 2):

Advertisement test is performed by using different values of following advertisement parameters.

Advertisement Type:

Advertisement types used by Testing Module are

  • Connectable Undirected: Advertisement type where any device can send scan response packet and connect request to the advertiser.

  • Non Connectable Undirected: Advertisement type where advertiser will not accept any connect request or scan request.

Advertisement Interval

Advertisement Interval specifies the intervening time between advertising messages. The advertisement interval used by Testing Module are 20ms, 100ms, 1000ms and 4000ms

Payload Size

Payload size is the size of the packet to be advertised. Payload of size 10 bytes and 30 bytes are advertised by Testing Module.

Transmit Power

Testing Module advertises packets with Tx power 0dbm and 4dbm.

Scanning Test (Application Id 3):

Scanning test is performed by using different values of following scanning parameters.

  • Scan for 3ms window in every 20ms interval.

  • Scan for 10ms window in every 10ms interval (full duty cycle).

Connection Test (Application Id 4)

Master node fill as many packets in the send queue as possible, each packet that was successfully put in the send queue is counted and MeshAccessConnections are used for sending these packets.

Latency Test (Application Id 5):

Packets between master and slave node keeps bouncing back and fourth until its time to live (ttl) is 0. Master node keeps track of time from start of packet sending until ttl is 0.

Terminal Commands

LED Test

starttest [bleAddress] led

This command will send a message to the slave node to start led test. The slave node will start its led test after it receives this message.

Example:

//Master node sends a message to node having ble address E3:8F:26:AD:82:DE
//to start led test.

starttest E3:8F:26:AD:82:DE led

Advertisement Test

starttest [bleAddress] adv [advInterval] [advTxPower] [advType] [advPayloadSize]

This command will send a message to the slave node to start advertisement test. The slave node will perform advertisement test for 30 secs after it receives this message.

Command Argument Value

bleAddress

Ble address of the slave node which will perform the test

advInterval

Advertising Interval in milli seconds (the value should be in the range 32-16384). It has a default value of 100

advTxPower

Transmit Power (the value should be either 0 or 4). It has a default value of 0.

advType

Advertisement Type (The values should be either 0 for connectable undirected or 3 for non connectable undirected). It has a default value of 0.

advPayloadSize

Payload Size (The value should be in the range 0 - 30 bytes). It has a default value of 10

Example:

//Master node sends a message to node having ble address E3:8F:26:AD:82:DE to start
//advertisement test with advertising interval 100ms, Tx Power 0 dbm, advertisement
//type connectable undirected and payload size 10 bytes.

starttest E3:8F:26:AD:82:DE adv 100 0 0 10

Scanning Test

starttest [bleAddress] scan [scanInterval] [scanWindow]

This command will send a message to the slave node to start scanning test. The slave node will scan for 30 secs after it receives this message.

Command Argument Value

bleAddress

Ble address of the slave node which will perform the test

scanInterval

Scanning Interval in milli seconds (the value should be in the range 4 - 16384). It has a default value of 20.

scanWindow

Scanning Window in milli seconds (the value should be either 4 or 16384). It has a default value of 3.

Example:

//Master node sends a message to node having ble address E3:8F:26:AD:82:DE to start
//scanning test with scan interval 20ms and scan window 20ms

starttest E3:8F:26:AD:82:DE scan 20 20

Latency Test

starttest [bleAddress] lat [ttl]

This command will send a message to the slave node that latency test should be performed. The master node starts the test after it receives acknowledgement message from slave.

Command Argument Value

bleAddress

Ble address of the slave node which will perform the test

ttl

Time to live (ttl) and has a default value of 30.

Example:

//Master node sends a message to node having ble address E3:8F:26:AD:82:DE to start
//latency test with ttl 20

starttest E3:8F:26:AD:82:DE lat 20

Connection Test

starttest [bleAddress] conn [connPacketSize]

This command will send a message to the slave node that connection test will be performed. The master node starts the test after it receives acknowledgement message from slave.

Command Argument Value

bleAddress

Ble address of the slave node which will perform the test

connPacketSize

Size of the packets to be queued. It has a default value of 12.

Example:

//Master node sends a message to node having ble address E3:8F:26:AD:82:DE to start
//connection test with connPacketSize 25
starttest E3:8F:26:AD:82:DE conn 25

All Test

starttest [bleAddress] all

This command will run each test sequentially for 30 seconds.Below you will find the tests covered in all tests.

Test Application Parameters

LED TEST

LATENCY TEST

ttl = 30

CONNECTION TEST

packetSize = 12, 36 bytes

SCANNING TEST

scanInterval = 20ms and scanWindow = 3ms, 20ms

ADVERTISEMENT TEST (advTxPower = 0 dbm, advType = Connectable)

advInterval = 100ms, 200ms and advPayloadSize = 30 bytes

ADVERTISEMENT TEST (advTxPower = 0 dbm, advType = Non-Connectable)

advInterval = 100ms, 400ms, 1000ms, 4000ms and advPayloadSize = 10, 30 bytes

ADVERTISEMENT TEST (advTxPower = 4 dbm, advType = Non-Connectable)

advInterval = 100ms and advPayloadSize = 30 bytes

Example:

//Master node sends a message to node having ble address E3:8F:26:AD:82:DE to
//start running tests sequentially each 30 sec

starttest E3:8F:26:AD:82:DE all
