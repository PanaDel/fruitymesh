ClcModule

General

The ClcModule contains the whole customer specific logic for regent. It works in conjunction with the ClcComm class that provides low level communication between FruityMesh and the regent controller.

Component_sense & Component_act Mapping

The component_sense and component_act messages are mapped to vendor specific data:

component_sense mapping

Generic data Vendor Specific data

nodeId

Dest. (Destination Node)

moduleId

CLC_MODULE_ID (150)

requestHandle

cnt (Counter)

actionType

CMD (UNSPECIFIED ⇒ Not Mapped, ERROR_RSP ⇒ C0, READ_RSP ⇒ C1, WRITE_RSP ⇒ C2)

component

modbus Adr. (Modbus Device Address)

registerAddress

HR Adr. (Holding Register Address)

payload

Data

The CR, length, Nr. HR, LF are automatically set or calculated from the data, the Source is automatically set using the nodeId of the sending device.

component_act mapping

Generic data Vendor Specific data

nodeId

Dest. (Destination Node)

moduleId

CLC_MODULE_ID (150)

requestHandle

cnt (Counter)

actionType

CMD (WRITE ⇒ D0, READ ⇒ D1, WRITE_ACK ⇒ D2)

component

modbus Adr. (Modbus Device Address)

registerAddress

HR Adr. (Holding Register Address)

payload

Either Data, or in case of READ mapped to Nr. HR

The CR, length, LF are automatically set or calculated from the data, the Source is automatically set using the nodeId of the sending device.
