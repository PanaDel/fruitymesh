Generating Source Code Distributions

General

A number of different distributions are supported. Each distribution only adds files and does not remove:

  • Github Distribution: Only open source code without any vendor code

  • Partner Distribution: Contains fruityloader and mway modules

  • Customer Distribution: Contains the customers specific source code as well

Source code or documentation in the vendor folder is included in the partner distribution, but source code and documentation in the vendor/mway folder is for internal use only.

Distributing

The srcdist.bat is used to generate all distributions, they will be generated in the parent directory of fruitymesh.

Ideally, you should have a repository for each distribution next to the fruitymesh directory.

  • fruitymesh_github

  • fruitymesh_partner

  • fruitymesh_waldmann

Every time, you want to distribute a new version of the source code, do the following steps:

  • Make sure that you have checked out the master branch in the distribution directories (e.g. fruitymesh_github, …​) or maybe you distribute in a special branch depending on the customer.

  • Execute the srcdist.bat script — The script will first clean the distribution directories but will omit the .git folder — It will then copy all appropriate files for the distribution

  • Commit the changes to the repository and push

Merging Changes

If there are changes made by the customer, you have to distribute him the latest master and get him to resolve all merge conflicts. Afterwards, pull his changes and copy them to your master branch and commit.
