VsModule

General

The VsModule contains the whole customer specific logic for vossloh. It works in conjunction with the VsComm class that provides low level communication between FruityMesh and the vossloh controller.

Component_sense & Component_act Mapping

The component_sense and component_act messages are mapped to vendor specific data:

component_sense mapping

Generic data Vendor Specific data

nodeId

Destination ID

moduleId

VS_MODULE_ID (151)

requestHandle

Packet ID (requestHandle)

actionType

Packet (UNSPECIFIED ⇒ PT_EVENT, ERROR_RSP ⇒ Not Mapped, READ_RSP ⇒ Not Mapped, WRITE_RSP ⇒ PT_RSP)

component

Byte 0: Mapped to inputNum for PT_EVENT, 0 for other Packet Types; Byte 1: Reserved 0; (Little Endian)

registerAddress

Byte 0: Mapped to Encryption Type; Byte 1: Mapped to evtId or cmdId depending on Packet Type; (Little Endian)

payload

Data (Without cmdId, evtId and inputNum)

Other data in the frame is set to fixed values or calculated.

component_act mapping

There is currently no mapping provided.
